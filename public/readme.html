Project

1. ファイル構成

アプリ本体は public/ にまとまっています。開発サーバーは Node.js 標準機能だけで静的ファイルを配信します。 

Basic_SaaS/
├── README.md
├── package.json
├── scripts/
│   └── dev-server.mjs
└── public/
    ├── index.html
    ├── readme.html
    ├── styles/
    │   ├── style.css
    │   └── readme.css
    └── scripts/
        ├── app.js
        └── readme.js

Run

2. クイックスタート

ビルドツールは不要です。次のコマンドで public/ を配信して、ブラウザで開きます。 

cd Basic_SaaS
npm run dev
# -> http://127.0.0.1:5500

VS Code の Live Server 拡張で public/index.html を開く運用もできます。

MVP

3. 仕様

F-01

名言入力

本文と著者を入力します。

F-02

プレビュー

入力に応じて Canvas がリアルタイム更新されます。

F-03

デザイン選択

テンプレート、背景色、文字色を選択します。

F-04

PNG 書き出し

1080 x 1080 の画像としてダウンロードします。

F-05

ローカル保存

LocalStorage に名言を永続化します。

F-06

ギャラリー

保存した名言を一覧、編集、削除できます。

F-07

シェア

Web Share API に対応し、非対応端末ではダウンロードに切り替えます。

受け入れ基準

  • テキストと著者を入力するとプレビューが即時更新される
  • PNG ダウンロードで 1080 x 1080 画像が保存できる
  • LocalStorage に 10 件以上保存して再表示できる
  • モバイル Chrome / Safari でシェア先に画像を渡せる

データモデル

キー quoteshare.quotes に、次のオブジェクト配列を JSON として保存します。

{
  id: string;
  text: string;
  author: string;
  template: 'minimal' | 'serif' | 'bold';
  bg: string;
  fg: string;
  thumbnail: string;
  createdAt: string;
  updatedAt: string;
}

Step 1

4.1 HTML / CSS / JavaScript

セマンティック HTML

header、main、section、footer を使い、画面構造をブラウザと支援技術に伝えます。

CSS カスタムプロパティ + Grid

色、角丸、余白を変数化し、画面幅に応じて 1 カラムと 2 カラムを切り替えます。

Canvas API

背景、装飾、本文、著者を 1080 x 1080 の Canvas に描画し、PNG へ変換します。

LocalStorage

小さな JSON データをブラウザに保存し、再訪問時にもギャラリーを復元します。

:root { --primary: #6366f1; --radius: 12px; }
.editor-grid {
  display: grid;
  grid-template-columns: minmax(0, 1fr) minmax(0, 380px);
  gap: 24px;
}
@media (max-width: 880px) {
  .editor-grid { grid-template-columns: 1fr; }
}

ユーザー入力を画面に出すときは innerHTML ではなく textContent を使います。

Step 2

4.2 外部 API を呼び出す

本 MVP は外部 API を使いませんが、SaaS を拡張するなら fetch、HTTP ステータス、CORS、API キー管理の理解が必要です。 

try {
  const res = await fetch('https://example.com/api/data', {
    headers: { Accept: 'application/json' },
  });
  if (!res.ok) throw new Error(`API error: ${res.status}`);
  const data = await res.json();
} catch (err) {
  console.error(err);
}
Code意味
200成功
400リクエスト不正
401未認証
403権限なし
404見つからない
429レート制限超過
500サーバーエラー

CORS

ブラウザは別オリジンの fetch を制限します。サーバーが Access-Control-Allow-Origin を返さないと JavaScript からレスポンスを読めません。

API キー

ブラウザの JavaScript は誰でも読めます。秘密情報はフロントに置かず、BFF やサーバーレス関数で扱います。

4.3 クラウド

種類サービス例用途
静的ホスティングFirebase Hosting / Cloudflare Pages / VercelHTML/CSS/JS を世界配信
認証Firebase Authentication / Auth0Google ログイン等
データベースCloud Firestore / Supabaseクラウド側にデータ保存・同期

静的ホスティングの基本フロー

npm install -g firebase-tools
firebase login
firebase init hosting
firebase deploy

クラウド DB を使う場合は、認証とセキュリティルールを必ずセットで設計します。 

match /users/{uid}/quotes/{doc} {
  allow read, write: if request.auth != null && request.auth.uid == uid;
}
